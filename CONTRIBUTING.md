# Contributing to N8N WooCommerce Integration

Thanks for your interest in contributing! Here's how you can help improve this plugin.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or fix
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Setup

### Requirements
- WordPress development environment
- WooCommerce installed and activated
- N8N instance for testing webhooks
- PHP 7.4 or higher

### Local Testing
1. Install the plugin in your WordPress test site
2. Set up a local N8N instance or use a test webhook service
3. Configure the webhook URL in plugin settings
4. Test various scenarios (orders, customers, status changes)

## Code Style

- Follow WordPress PHP coding standards
- Use meaningful variable and function names
- Add comments for complex logic (keep it natural, not robotic)
- Keep functions focused and single-purpose

## What to Contribute

### Bug Reports
- Check if the issue already exists
- Provide clear reproduction steps
- Include WordPress, WooCommerce, and PHP versions
- Share relevant error logs

### Feature Requests
- Explain the use case clearly
- Show how it benefits users
- Consider if it fits the plugin's scope

### Code Contributions
- Write clean, readable code
- Test your changes thoroughly
- Update documentation if needed
- Keep commits focused and descriptive

## Areas That Need Help

- Additional event types (products, coupons, etc.)
- Translation support
- Performance optimizations
- Documentation improvements
- UI/UX enhancements

## Testing Checklist

Before submitting a pull request:

- [ ] Plugin activates without errors
- [ ] Settings page loads correctly
- [ ] Webhook test button works
- [ ] Events trigger correctly
- [ ] Data format is valid JSON
- [ ] Individual webhooks work as expected
- [ ] No PHP warnings or notices
- [ ] Compatible with latest WooCommerce
- [ ] Works with HPOS enabled

## Questions?

Feel free to open an issue for:
- Implementation questions
- Architecture discussions
- Feature proposals
- General feedback

## Code of Conduct

- Be respectful and constructive
- Focus on the code, not the person
- Help others learn and improve
- Keep discussions relevant
